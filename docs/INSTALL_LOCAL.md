# Send Guard for Gmail インストール・初期設定ガイド

このドキュメントでは、Chrome拡張機能「Send Guard for Gmail」をご利用のブラウザ（Google Chrome）にインストールし、使えるようにするまでの手順を解説します。Send Guardは独立したプロジェクトであり、Googleとの提携、承認、後援関係はありません。Gmail™はGoogle LLCの商標です。

---

## 1. Chromeへのインストール（読み込み）

拡張機能は現在ローカルフォルダにあるため、「デベロッパーモード」を使って手動で読み込みます。

1. Chromeブラウザを開き、アドレスバーに `chrome://extensions/` と入力してEnterを押します。
2. 画面の右上にある **「デベロッパーモード」** のスイッチを **ON** にします。
3. 左上に現れる **「パッケージ化されていない拡張機能を読み込む」** ボタンをクリックします。
4. ファイル選択ダイアログが開くので、このプロジェクトの **`extension`** フォルダを選択して「フォルダの選択」をクリックします。
5. 拡張機能一覧に **「Send Guard for Gmail」** が表示されればインストール完了です。

開発時に読み込む正典ソースは **`extension/` 配下のみ** です。提出前は `npm run release:package` を実行し、生成される `extension_package/extension/` と提出ZIPを使います。`preview/` は設計確認用モックであり、本番拡張パッケージには含めないでください。

---

## 2. 初期設定を行う（推奨）

インストール直後はCC/BCCリストなどが空のため、一度設定画面を開いて初期設定を行うことをおすすめします。

1. Chrome画面右上の 🧩（拡張機能）アイコンをクリックし、リストから **Send Guard for Gmail** を見つけます。
2. 拡張機能の名前の横にある **「⋮（縦の三点リーダー）」** をクリックし、**「オプション」** を選択します。（※設定画面が開きます）
3. **テーマ設定**: 右上のセレクトボックスからお好みのテーマ（システム・ライト・ダーク）を選びます。
4. **表示言語**: Auto / 日本語 / English から選択します。言語設定は `chrome.storage.local` に保存され、外部翻訳サービスやリモート翻訳ファイルは使用しません。
5. **自動CC/BCC設定**: 必要に応じて「自動CCを有効にする」「自動BCCを有効にする」をONにします。どちらもメールアドレスを入力し、Enter・カンマ・スペースで確定するとチップとして登録されます。複数アドレスの貼り付けにも対応し、重複は登録されません。登録済みCC/BCCアドレスは各チップの×で1件ずつ削除できます。追加・削除は `chrome.storage.local` へ即時保存されます。
6. 画面右下の **「設定を保存」** ボタンをクリックします。

---

## 3. Gmailでの動作確認

設定が完了したら、実際にGmailで動くかテストしてみましょう。

1. [Gmail (https://mail.google.com/)](https://mail.google.com/) を開き、再読み込み（F5 または Ctrl+R）します。
2. **「作成」** ボタンを押して新規メール作成画面を開きます。
3. 宛先、件名、本文などを入力します（意図的に件名を空にしたり、本文に「添付」とだけ書いてファイルは付けない等でテストすると分かりやすいです）。
4. **「送信」** ボタンを押すか、本文入力中に **Ctrl+Enter / Cmd+Enter** を押します。
5. 画面中央に **「送信前チェック」のポップアップ（モーダル）が割り込んで表示される** ことを確認します。
6. モーダル内で警告内容（またはすべてOKであること）を確認し、「キャンセル」で戻るか、「送信する」ボタンで実際にメールを送信できるかを確認します。
7. 設定画面で日本語/Englishを切り替え、設定画面と送信前モーダルの表示がreloadなしで切り替わることを確認します。
8. 件名、宛先ドメイン、添付ファイル名、添付サイズなどGmail由来データが翻訳されず、そのまま安全に表示されることを確認します。

## 4. 提出候補ZIPの作成

開発依存関係をインストール済みの状態で、以下を実行します。

```bash
npm run release:package
npm run release:validate
npm run release:reproducibility
```

この処理は `extension/` を正典として、公開リポジトリ用の `public_repo/extension/` と読み込み用ステージング `extension_package/extension/` をホワイトリスト方式で再生成します。提出候補ZIPはNode.jsの決定的STORE方式で `release/artifacts/` に生成され、ZIPのルート直下に `manifest.json` があること、directory entryが0件であること、全entryが許可一覧と完全一致すること、checksumファイルのSHA-256が実ZIPと一致することを検査します。再現性コマンドは同一実行内で2つのZIPを独立生成し、バイト列とSHA-256を比較します。ZIPに `README`、`docs`、`tests`、`tools`、`package.json`、`node_modules` を手動で追加しないでください。

設定画面はライト/ダークテーマ、360px相当までの狭いデスクトップウィンドウ、長いアドレス/キーワードの折り返し、キーボード操作、reduced-motion設定、`aria-live`による保存通知に対応しています。Chrome 150 + Gmail Webの実機確認はこのリポジトリ内の自動テストでは代替できず、公開前に別途必要です。

## 💡 トラブルシューティング

- **Q. モーダルが表示されない、またはCC/BCCが自動挿入されない**
  - **A.** Gmailの画面構造（DOM）がGoogle側でアップデートされた可能性があります。`docs/ENTERPRISE_REVIEW.md` の既知の制限を確認し、必要に応じて `extension/content.js` や `extension/gmail_dom.js` のセレクタ（要素の検索条件）を更新してください。
- **Q. 設定を変更したのに反映されない**
  - **A.** 保存した設定は、開いているGmailタブにも再スキャンで反映されます。拡張機能のコード自体を書き換えた場合は `chrome://extensions/` で該当拡張機能の「🔄（再読み込み）」ボタンを押し、その後Gmailを再読み込みしてください。
- **Q. 英語UIにしても添付キーワード「添付」が残る**
  - **A.** v0.1では固定キーワード「添付」を維持しています。UIは日本語/英語に切り替わりますが、Gmail由来データや固定添付判定ロジックは翻訳しません。
